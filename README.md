# 브라우저로 개발 환경 만들기 (Codespaces)

회사 노트북의 보안 정책 등으로 **내 컴퓨터에 설치가 불가능한 분들을 위한 비상 경로**입니다. 설치가 가능하다면 강의의 한 줄 설치 명령(macOS/Windows)을 쓰는 것이 기본입니다 — 그쪽이 내 컴퓨터에 모든 것이 남아 다루기 쉽습니다.

> **준비물**: 강의에서 받은 인증코드. 로컬에서 설치하는 수강생과 **똑같은 설치 과정**을 브라우저 안에서 진행합니다 — 단지 내 컴퓨터 대신 클라우드(Codespace)에서 돌 뿐입니다.

## 시작하기

1. GitHub에 로그인한 상태에서, 이 페이지 위쪽의 **Code → Codespaces → Create codespace on main**을 누릅니다.
2. 몇 분 기다리면 브라우저에 VS Code 화면이 뜹니다(이 동안 공통 도구가 미리 설치됩니다).
3. 상단 메뉴 **Terminal → New Terminal** 로 터미널을 열고, 아래 **한 줄을 붙여넣고 Enter**:

   ```
   curl -fsSL https://saas-starter-setup.vercel.app/setup.sh | bash
   ```

   그러면 차례로 묻습니다 — **인증코드**(강의에서 받은 것) → **프로젝트 이름**(예: `my-service`) → **코딩 에이전트**(엔터 = Claude Code). 그다음 서비스 로그인(브라우저 승인)·GitHub 백업·환경 점검(doctor)까지 자동으로 이어집니다. **로컬에서 설치하는 수강생과 똑같은 과정**입니다.
4. 끝에 안내되는 대로 프로젝트 폴더로 이동해 코딩 에이전트를 실행하고(`cd <이름>` → `claude`), 구독 계정으로 로그인한 뒤 **"START-HERE.md 읽고 시작하자"**라고 말하면 됩니다.
5. 환경 상태가 궁금하면 언제든 프로젝트 폴더에서 `pnpm run doctor`.

> Codespace 기본 토큰은 내 계정에 새 레포를 만들 권한이 없을 수 있습니다. 그러면 GitHub 백업 단계에서 "`gh auth login` 후 다시"라는 안내가 뜨는데, 그대로 `gh auth login`을 한 번 한 뒤 `node scripts/connect-repos.mjs`를 다시 실행하면 됩니다(doctor도 이 상태를 짚어 줍니다).

## 내 작업물은 어디에 남나

- 설치 마지막의 **GitHub 백업** 단계에서, 내 작업이 내 계정의 private 레포 3개(`<이름>-workspace`·`<이름>-web`·`<이름>-mobile`)에 올라갑니다. **그때부터는 Codespace가 사라져도 작업이 내 GitHub에 안전합니다** — 이게 클라우드 경로에서 가장 중요한 부분입니다.
- 백업이 됐는지 `pnpm run doctor`로 확인하세요. (토큰 권한으로 막혔다면 위 *시작하기*의 `gh auth login` 안내를 따르면 됩니다.)
- 이후 `/go-live` 단계가 그 위에 Vercel·Supabase·Sentry를 붙여 첫 배포까지 잇습니다 — 코드와 서비스가 전부 내 소유의 클라우드에 남습니다.
- GitHub은 30일간 쓰지 않은 Codespace를 자동 삭제합니다. 하지만 백업만 돼 있으면 삭제돼도 그 3개 레포에서 다시 시작할 수 있습니다.

## 알아둘 것

- **다음에 다시 올 때는 새로 만들지 말고 이어서 여세요** — Code → Codespaces 목록에 이미 만든 것이 보입니다. 그걸 클릭하면 작업하던 그대로 열립니다(설치는 이미 끝났으니 다시 묻지 않습니다). 새로 만들면 빈 환경이 하나 더 생겨 "내 작업이 사라진" 것처럼 보입니다.
- GitHub 무료 계정은 Codespaces를 월 120 코어시간(2코어 머신 기준 60시간)까지 무료로 쓸 수 있습니다. 쓰지 않을 때는 Codespace를 Stop 해 두세요.
- 이 레포 자체에는 코드가 거의 없습니다 — Codespace를 만들 때 환경을 꾸며 주는 설정(`.devcontainer/`)만 있습니다.
